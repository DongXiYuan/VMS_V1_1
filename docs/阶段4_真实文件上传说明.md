# VMS 阶段 4：真实文件上传说明

## 一、启动与访问

打开 PowerShell：

```powershell
powershell -ExecutionPolicy Bypass -File D:\VMS_V1\backend\run_backend.ps1
```

浏览器访问：

```text
http://127.0.0.1:8000/
```

如果旧服务仍在运行，请先在旧 PowerShell 窗口按 `Ctrl+C`。

## 二、导入资产表

1. 左侧点击“文件导入”。
2. 在“资产表上传”区域选择 `.xls` 或 `.xlsx` 文件。
3. 点击“上传并预览”。
4. 检查新增、更新、未变化和异常数量。
5. 确认无误后点击“确认发布”。

上传预览不会立即修改资产库。只有点击“确认发布”后，解析成功的数据才会写入资产库。

## 三、导入漏洞扫描文件

1. 在“漏洞扫描文件上传”区域选择扫描器。
2. 填写月份，例如 `2026-05`。
3. 青藤云和阿里云选择 `.xls` 或 `.xlsx` 文件；绿盟选择 `.zip` 文件。
4. 点击“上传并预览”。
5. 检查有效漏洞数、低危过滤数、重复合并数和问题明细。
6. 确认后点击“确认发布”。

绿盟 ZIP 会先读取 `index.xls`，仅打开中危和高危数量大于 `0` 的主机报告。

## 四、错误分级

| 级别 | 页面行为 | 常见情况 |
|---|---|---|
| `blocking` | 禁止发布，需要重新上传 | 文件损坏、缺少必填工作表、缺少必填列、绿盟缺少 `index.xls` |
| `warning` | 再次确认后允许发布 | IP 未匹配资产、危险等级异常、绿盟缺少部分主机报告、同月替换已有批次 |

## 五、同月替换

同一扫描器、同一月份再次上传修订文件时，页面会提示替换当前有效批次。旧批次保留历史，新批次发布后生效。已经人工修改的处置状态和备注不会被覆盖。

## 六、预览有效期

未发布预览保留 `24` 小时。过期后系统会删除临时文件，需要重新上传。正式发布后的原始文件进入本机留存目录。

## 七、GitHub 安全规则

以下目录只保存在本机，不上传 GitHub：

```text
samples/
archives/
outputs/
backend/data/
test-data/
```

自动化测试源代码会纳入版本控制，但测试 Excel、ZIP、数据库和运行报告不会上传。
