# VMS 阶段 2 中文接口文档

## 一、用途

阶段 2 用于验证后端最小闭环：

```text
导入资产样例
  ↓
导入三类扫描器漏洞样例
  ↓
查询标准漏洞清单
  ↓
修改处置状态和备注
  ↓
查看修改历史和导入异常
```

阶段 3 已增加直观的网页业务入口。本文档继续作为接口排查说明保留。

## 二、启动与访问

打开 PowerShell，运行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\VMS_V1\backend\run_backend.ps1
```

浏览器访问：

```text
http://127.0.0.1:8000/
```

该地址现在显示阶段 3 网页业务界面。

需要直接测试接口时，浏览器访问：

```text
http://127.0.0.1:8000/docs
```

Swagger 中每个接口右侧有一个展开按钮。展开后点击 `Try it out`，再点击 `Execute` 即可执行。

## 三、推荐操作顺序

### 第 1 步：检查服务状态

接口：

```text
GET /api/health
```

用途：确认后端是否正常启动。

预期返回：

```json
{
  "status": "ok"
}
```

### 第 2 步：导入资产表样例

接口：

```text
POST /api/dev/import-assets-sample
```

用途：读取 `samples\资产表（样例）.xlsx`，导入 IP、组织、项目、工作空间和负责人。

无需填写参数。

预期返回：

```json
{
  "created": 0,
  "updated": 7,
  "total": 7
}
```

首次执行时 `created` 通常为 `7`。重复执行时 `updated` 通常为 `7`。

### 第 3 步：导入漏洞样例

接口：

```text
POST /api/dev/import-vulnerabilities-sample
```

用途：读取青藤云、阿里云和绿盟样例，生成标准漏洞记录。

请求内容：

```json
{
  "scan_month": "2026-05"
}
```

预期结果：

```text
标准漏洞记录：13
青藤云：5
阿里云：7
绿盟：1
```

### 第 4 步：查看资产列表

接口：

```text
GET /api/assets
```

用途：确认资产表已经成功导入。

预期结果：

```text
total = 7
```

### 第 5 步：查看标准漏洞清单

接口：

```text
GET /api/records
```

用途：查看标准漏洞清单。

可选筛选条件：

| 参数 | 示例 | 说明 |
|---|---|---|
| `scan_month` | `2026-05` | 扫描月份 |
| `scanner_type` | `绿盟` | 扫描器类型 |
| `project` | 项目名称 | 系统名称 |
| `handle_status` | `待修复` | 当前处置状态 |

不填写参数时，返回全部记录。

每条记录中的 `id` 是漏洞记录 ID。修改状态时需要使用它。

### 第 6 步：修改处置状态和备注

接口：

```text
PATCH /api/records/{record_id}
```

用途：修改指定漏洞的处置状态和备注。

路径参数：

```text
record_id
```

填写第 5 步查询到的漏洞记录 ID，例如：

```text
1
```

请求内容：

```json
{
  "handle_status": "已通知",
  "remark": "等待反馈",
  "changed_by": "admin"
}
```

### 第 7 步：查看修改历史

接口：

```text
GET /api/records/{record_id}/changes
```

用途：查看指定漏洞的状态和备注修改历史。

路径参数填写刚才修改过的漏洞记录 ID。

### 第 8 步：查看导入异常

接口：

```text
GET /api/anomalies
```

用途：查看最新月份有效批次中的导入异常。

当前样例预期有 6 条异常。原因是绿盟索引中存在主机，但样例目录没有提供对应的主机报告。

可以填写：

```text
scan_month = 2026-05
```

也可以留空，系统会自动查询最新月份。

## 四、阶段 2 接口总览

| 分组 | 方法 | 地址 | 用途 |
|---|---|---|---|
| 系统状态 | GET | `/api/health` | 检查服务 |
| 样例导入 | POST | `/api/dev/import-assets-sample` | 导入资产样例 |
| 样例导入 | POST | `/api/dev/import-vulnerabilities-sample` | 导入漏洞样例 |
| 资产管理 | GET | `/api/assets` | 查看资产 |
| 漏洞清单 | GET | `/api/records` | 查询漏洞 |
| 漏洞清单 | PATCH | `/api/records/{record_id}` | 修改状态和备注 |
| 漏洞清单 | GET | `/api/records/{record_id}/changes` | 查看变更历史 |
| 异常处理 | GET | `/api/anomalies` | 查看异常 |

## 五、说明

- `/api/dev/*` 接口仅用于阶段 2 本机样例体验。
- 后续会替换为真实文件上传页面。
- 当前尚未实现登录，`changed_by` 暂时手动填写。
- 最终网页会提供按钮、表格和筛选框，不需要直接操作 Swagger。
