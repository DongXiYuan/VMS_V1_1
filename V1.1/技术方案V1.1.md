# VMS_V1 技术方案 V1.1

## 1. 方案定位

`V1.1` 建立在 `V1.0` 已有真实上传、标准清单、状态维护、统计和导出能力之上，不改变当前系统整体形态，继续采用：

- FastAPI 后端
- SQLite 本地数据库
- 原生 HTML / CSS / JavaScript 前端

本次技术方案聚焦在四个子系统增强：

1. 批量操作子系统
2. 异常资产处理子系统
3. 数据分析子系统
4. 复发漏洞识别与提醒子系统

## 2. 设计原则

### 2.1 延续现有架构

`V1.1` 不引入新框架，不重做前端，不切换数据库。

### 2.2 以单管理员使用为前提

当前版本默认由管理员本人试用，因此先实现高价值业务功能，不优先做登录、权限与复杂协作控制。

### 2.3 保留追溯能力

涉及批量修改、批量删除、重匹配等动作时，必须保留审计痕迹。

### 2.4 删除不等于消失

对业务记录采用软删除，保证可追溯、可解释、可核查。

## 3. 功能架构

## 3.1 模块划分

### A. 批量操作模块

- 批量选中漏洞
- 批量修改处置状态
- 批量软删除

### B. 异常处理模块

- 未匹配资产漏洞查询
- 未匹配 IP 导出
- 去重 IP 导出
- 批量重匹配

### C. 统计分析模块

- Top 10 系统
- Top 10 漏洞名称
- 近 6 个月趋势分析
- 修复率分析
- 复发漏洞趋势

### D. 复发提醒模块

- 复发漏洞识别
- 复发高亮
- 复发筛选
- 复发统计

## 4. 数据模型设计

## 4.1 漏洞记录表扩展

建议在 `vulnerability_records` 基础上补充以下字段：

```text
is_deleted
deleted_at
deleted_by
deleted_reason
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `is_deleted` | bool | 是否软删除 |
| `deleted_at` | datetime | 删除时间 |
| `deleted_by` | string | 删除人 |
| `deleted_reason` | text | 删除原因 |

说明：

- 当前查询默认 `is_deleted = false`
- 统计默认排除 `is_deleted = true`
- 导出默认排除 `is_deleted = true`

## 4.2 复发识别逻辑

复发漏洞不建议单独永久存储为静态字段，建议优先采用动态计算。

### 复发判定规则

某条本月漏洞满足以下条件时，判定为“复发待复查”：

1. 存在上一月同扫描器、同 IP、同端口、同归一化漏洞名称记录
2. 上月处置状态属于：
   - 已修复
   - 主机已关机
   - 采用缓解措施
3. 本月该漏洞再次出现

### 判定结果输出

接口返回新增字段：

```text
is_reopened
reopened_from_status
```

## 4.3 批量操作审计

沿用现有 `record_changes`，并在批量操作时逐条写入。

### 批量修改状态

每条记录写入：

```text
record_id
field_name = handle_status
old_value
new_value
changed_by
changed_at
```

### 批量软删除

建议同时记录两类信息：

1. 漏洞表自身删除字段
2. 审计记录

审计记录可写为：

```text
field_name = is_deleted
old_value = false
new_value = true
```

## 5. 接口设计

## 5.1 批量修改状态

新增接口建议：

```text
POST /api/records/batch/status
```

请求体：

```json
{
  "record_ids": [1, 2, 3],
  "handle_status": "已通知",
  "changed_by": "admin"
}
```

返回：

- 成功条数
- 失败条数
- 失败记录明细

## 5.2 批量软删除

新增接口建议：

```text
POST /api/records/batch/delete
```

请求体：

```json
{
  "record_ids": [1, 2, 3],
  "deleted_reason": "不需要修复",
  "deleted_by": "admin"
}
```

返回：

- 成功条数
- 删除原因
- 删除时间

## 5.3 未匹配 IP 导出

新增接口建议：

```text
GET /api/anomalies/unmatched-assets/export
```

参数建议：

```text
scan_month_from
scan_month_to
scanner_type
dedup_by_ip=true|false
```

返回：

- Excel 文件下载

## 5.4 异常重匹配

新增接口建议：

```text
POST /api/anomalies/unmatched-assets/rematch
```

说明：

- 可按筛选结果批量执行
- 重匹配成功后回写漏洞记录资产信息

## 5.5 统计接口增强

建议在现有统计接口基础上增加子接口，避免一个接口承载过多结构。

例如：

```text
GET /api/statistics/top-projects
GET /api/statistics/top-vulnerabilities
GET /api/statistics/monthly-trend
GET /api/statistics/reopened-trend
```

## 6. 前端设计

## 6.1 漏洞清单页改造

新增内容：

1. 每行前增加选择框
2. 表头增加全选框
3. 页面顶部增加批量操作栏
4. 支持批量修改状态
5. 支持批量删除
6. 复发漏洞高亮样式
7. 筛选区增加“仅看复发漏洞”

### 高亮设计建议

对复发漏洞增加：

- 淡红或淡橙背景
- `复发待复查` 标签

设计目标：

- 一眼看出需要重点复核的记录
- 不影响原有列表可读性

## 6.2 异常处理页改造

新增区域：

1. 未匹配资产异常列表
2. 导出全部未匹配 IP
3. 导出去重 IP
4. 批量重匹配按钮

## 6.3 统计页改造

新增图表和卡片：

1. 系统漏洞总数 Top 10
2. 漏洞名称次数 Top 10
3. 近 6 个月漏洞总数折线图
4. 近 6 个月修复率折线图
5. 复发漏洞数量趋势图
6. 扫描器趋势图

## 7. 统计口径设计

## 7.1 系统漏洞总数 Top 10

统计当前筛选范围内，每个项目的漏洞数量，按降序排序取前 10。

## 7.2 漏洞名称 Top 10

统计当前筛选范围内，归一化漏洞名称的出现次数，按降序排序取前 10。

## 7.3 漏洞总数趋势

按月统计近 6 个月有效漏洞总数。

说明：

- 排除软删除记录
- 排除未发布预览

## 7.4 修复率趋势

按月统计：

```text
修复率 = 已处置数 / 总漏洞数
```

已处置数口径：

- 已修复
- 主机已关机
- 采用缓解措施

## 7.5 复发漏洞趋势

按月统计满足复发判定条件的漏洞数量。

## 8. 处理流程设计

## 8.1 批量修改状态流程

```text
勾选记录
  ->
点击批量修改状态
  ->
选择目标状态
  ->
确认提交
  ->
逐条更新状态并写入变更审计
  ->
刷新列表和统计
```

## 8.2 批量软删除流程

```text
勾选记录
  ->
点击批量删除
  ->
填写删除原因
  ->
二次确认
  ->
逐条写入软删除字段和审计记录
  ->
前台默认隐藏
  ->
统计和导出自动排除
```

## 8.3 异常重匹配流程

```text
更新资产表
  ->
进入异常处理页
  ->
筛选未匹配记录
  ->
点击批量重匹配
  ->
重新按 IP 匹配资产
  ->
成功则回写资产信息
  ->
失败则保留异常状态
```

## 9. 风险与约束

## 9.1 软删除带来的风险

风险：

- 用户误以为数据真的不存在

控制方式：

- 页面增加“已删除记录默认隐藏”的说明
- 后续可补充“查看已删除记录”模式

## 9.2 批量操作带来的风险

风险：

- 批量误操作影响多条记录

控制方式：

- 二次确认
- 明确显示本次影响条数
- 审计可追溯

## 9.3 统计口径误解风险

风险：

- 修复率和处置率容易混淆

控制方式：

- 图表和卡片旁明确展示统计口径

## 10. 实施建议

建议 `V1.1` 分三步落地：

### 第一步：批量操作与软删除

- 批量选中
- 批量改状态
- 批量软删除

### 第二步：异常处理增强

- 导出未匹配 IP
- 去重导出
- 批量重匹配

### 第三步：统计与复发高亮

- 新增统计接口
- 统计图表
- 复发漏洞高亮和筛选

## 11. V1.1 完成标志

当以下能力全部落地并可实测时，可认为 `V1.1` 达标：

1. 漏洞列表支持批量改状态
2. 漏洞列表支持批量软删除
3. 软删除后记录不参与导出和统计
4. 未匹配 IP 可批量导出并去重
5. 异常记录支持重匹配
6. 统计页新增 Top 10 和趋势图
7. 上月已处置但本月复发的漏洞可高亮并筛选
