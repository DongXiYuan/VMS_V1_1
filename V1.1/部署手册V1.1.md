# VMS_V1_1 部署手册 V1.1

## 1. 适用范围

本手册适用于 `VMS_V1_1` 独立项目版本，面向你当前计划使用的 VMware 虚拟机场景。

结合当前项目代码结构和启动方式，本版本支持两种部署思路：

1. `Windows Server` 虚拟机部署（推荐）
2. `Linux` 虚拟机部署（可用，但需手动启动）

推荐 `Windows Server` 的原因：

- 当前项目自带 `PowerShell` 启动脚本
- 与你当前本地使用习惯一致
- 后续排障和备份更直接

---

## 2. 部署建议

### 2.1 推荐部署方案

建议优先使用：

- 虚拟机系统：`Windows Server 2019` 或 `Windows Server 2022`
- 部署目录：`D:\VMS_V1_1`
- 访问方式：局域网访问
- 运行方式：常驻启动

### 2.2 最低资源建议

结合你当前环境，建议如下：

| 项目 | 建议 |
|---|---|
| CPU | 4 核及以上 |
| 内存 | 8 GB 起步，推荐 16 GB |
| 磁盘 | 可用空间 30 GB 以上，推荐 100 GB |
| Python | 3.12 |
| 网络 | 虚拟机固定 IP 或 DHCP 保留地址 |

## 4. Windows Server 部署步骤（推荐）

## 4.1 安装 Python

在虚拟机中安装 `Python 3.12`，安装完成后打开 `PowerShell` 验证：

```powershell
python --version
pip --version
```

预期结果：

- 能输出 Python 版本
- 能输出 pip 版本

## 4.2 获取项目代码

你可以使用以下任一方式：

### 方式一：直接从 GitHub 下载

仓库地址：

```text
https://github.com/DongXiYuan/VMS_V1_1
```

下载 ZIP 后解压到：

```text
D:\VMS_V1_1
```

### 方式二：使用 git 克隆

```powershell
git clone https://github.com/DongXiYuan/VMS_V1_1.git D:\VMS_V1_1
```

## 4.3 安装依赖

进入项目目录执行：

```powershell
cd D:\VMS_V1_1
pip install -r .\backend\requirements.txt
```

## 4.4 首次启动

执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\backend\run_backend.ps1
```

默认监听：

```text
http://127.0.0.1:8000/
```

接口文档：

```text
http://127.0.0.1:8000/docs
```

## 4.5 局域网访问方式

如果希望局域网其他机器访问，不要使用默认脚本，而是手动启动：

```powershell
cd D:\VMS_V1_1\backend
$env:PYTHONPATH="D:\VMS_V1_1\backend;D:\VMS_V1_1\prototype\src"
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

然后通过虚拟机 IP 访问，例如：

```text
http://192.168.1.50:8000/
```

## 4.6 防火墙放行

如果虚拟机开启了 Windows 防火墙，需要放行 `8000` 端口：

```powershell
New-NetFirewallRule -DisplayName "VMS_V1_1_8000" -Direction Inbound -Protocol TCP -LocalPort 8000 -Action Allow
```

## 5. Linux 虚拟机部署步骤（备选）

适用于 Ubuntu / Debian / Rocky Linux 等系统。

## 5.1 安装系统依赖

以 Ubuntu 为例：

```bash
sudo apt update
sudo apt install -y python3 python3-pip git
```

## 5.2 获取项目代码

```bash
git clone https://github.com/DongXiYuan/VMS_V1_1.git /opt/VMS_V1_1
```

## 5.3 安装 Python 依赖

```bash
cd /opt/VMS_V1_1
pip3 install -r backend/requirements.txt
```

## 5.4 启动服务

```bash
cd /opt/VMS_V1_1/backend
export PYTHONPATH="/opt/VMS_V1_1/backend:/opt/VMS_V1_1/prototype/src"
python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

访问地址：

```text
http://<虚拟机IP>:8000/
```

## 6. 启动验证

服务启动成功后，建议依次检查：

1. 首页是否可打开
2. Swagger 接口文档是否可打开
3. 资产上传是否可预览
4. 漏洞上传是否可预览
5. 工作台是否正常显示

验证地址：

```text
http://127.0.0.1:8000/
http://127.0.0.1:8000/docs
```

如果是局域网部署，则把 `127.0.0.1` 替换为虚拟机 IP。

## 9. 版本升级建议

从后续版本升级时，建议按以下顺序操作：

1. 停止当前服务
2. 备份 `backend/data/`
3. 更新项目代码
4. 如依赖有变化，重新执行依赖安装
5. 启动服务
6. 验证上传、查询、导出、统计是否正常

## 10. 常见问题

### 10.1 页面打不开

先检查服务是否真的已启动：

```powershell
netstat -ano | findstr 8000
```

### 10.2 局域网能启动但其他机器访问不到

优先检查：

1. 是否使用了 `--host 0.0.0.0`
2. 虚拟机防火墙是否放行 `8000`
3. VMware 网络模式是否允许局域网访问

### 10.3 上传后无数据

优先检查：

1. 资产表是否已正确导入并发布
2. 漏洞文件是否使用系统支持的格式
3. 预览页面是否存在阻断错误

## 11. 当前版本限制

当前 `V1.1` 独立项目版本仍有以下限制：

- 未提供登录与权限体系
- 未提供 Docker Compose 部署方式
- 未提供 HTTPS 配置
- 未提供 PostgreSQL / Redis 切换
- 趋势统计当前以表格展示为主

如果你后面准备把它长期稳定运行在虚拟机里，下一步最值得补的是：

1. 开机自启
2. 定时备份
3. 已删除记录管理入口
4. 统计图形化展示
